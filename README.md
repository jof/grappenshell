# grappenshell

A fake Linux shell served over SSH via [Tailscale](https://tailscale.com). Users SSH in and get what looks like a perfectly normal Linux system — except every command output is generated by an LLM. The filesystem, processes, logs, and system state are all hallucinated on the fly, maintaining just enough consistency to be *plausibly real*.

Built for April Fools' Day.

## How it works

Grappenshell runs as a [tsnet](https://tailscale.com/kb/1244/tsnet) service, appearing as a node on your tailnet. When someone SSHs in, they get a terminal session with a shell prompt. Commands are parsed and sent to an OpenAI-compatible LLM API (also accessed over Tailscale), which generates realistic-looking output — `ls` lists believable files, `cat` shows plausible contents, `ps aux` returns a reasonable process table, etc.

The LLM maintains conversation context across commands within a session, so the hallucinated system stays internally consistent (mostly).

## Architecture

```
cmd/grappenshell/       Entry point
internal/
  ssh/server.go         Tailscale tsnet SSH server (no auth — Tailscale handles it)
  shell/session.go      Interactive terminal session with LLM-backed command handling
  shell/parser.go       Shell-like command line tokenizer
  llm/client.go         LLM client interface (OpenAI-compatible API)
```

## Configuration

| Flag | Default | Description |
|------|---------|-------------|
| `-hostname` | `grappenshell` | Tailscale hostname for the node |
| `-system-prompt` | *(built-in)* | System prompt controlling the LLM's shell persona |

The LLM API endpoint is at `http://ai.example-tailnet.ts.net/v1` (accessible over Tailscale).

## Building & Running

```bash
go build -o grappenshell ./cmd/grappenshell
./grappenshell -hostname grappenshell
```

Then from any machine on the tailnet:

```bash
ssh grappenshell
```
